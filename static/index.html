          POST /session/diarize — produces a speaker-labelled
          transcript suitable as the "source of truth" for downstream
          features (summarizer, note-taker, speech-to-speech). Uses the
          CPU-based diarize backend by default (no token
          required); falls back to pyannote automatically
          if the result collapses to one speaker. Override via the
          DIARIZATION_BACKEND env var.
          


          For best results: if you know the speaker count
          (e.g. a 2-person podcast), set Min speakers and
          Max speakers explicitly — both backends use them as
          hard hints and produce dramatically cleaner turn boundaries
          (especially when a host returns after a long gap). Leave
          Language blank for mixed-language audio (auto-detect).
